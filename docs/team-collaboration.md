# 团队协作说明

这份文档适合 6 人课程大作业团队使用。

## 1. 推荐 Git 分工方式

- `main`：稳定主分支
- 每个人单独开自己的功能分支
- 开工前先拉最新代码
- 大改动合并前先做代码检查

## 2. 推荐的 6 人职责划分

- 组员 1：玩家移动与基础操作
- 组员 2：战斗、技能、碰撞交互
- 组员 3：敌人系统、AI、状态逻辑
- 组员 4：分数、背包、任务、规则系统
- 组员 5：UI、场景搭建、Godot 脚本和资源整合
- 组员 6：构建脚本、联调、冲突处理、最终测试

## 3. 协作时尽量遵守这些规则

- 尽量不要多人同时改同一个 `.cpp` 文件
- 新系统优先新建文件
- 类名、方法名、文件名保持统一风格
- 新增流程时同步更新文档

## 4. 每次提交前建议检查

- 是否能成功编译 debug 版
- `demo/` 工程是否还能正常打开
- GDExtension 是否加载成功
- 新写的 C++ 方法是否能在 Godot 中调用
- `demo/coursework_extension.gdextension` 是否仍对应正确产物

## 5. 推荐的开发节奏

1. 先分模块
2. 再分文件负责人
3. 每个人在自己的模块内开发
4. 每天或每阶段统一合并一次
5. 每次合并后立刻编译和运行 demo 验证
