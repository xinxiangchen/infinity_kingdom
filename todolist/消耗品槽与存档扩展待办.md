# 消耗品槽与存档扩展待办

更新日期：2026-06-11

本文档用于固定两个后续必须补的系统：主动消耗品槽、复杂存档。技能分支成长第一版暂不进入待办，不作为近期目标。

## 1. 第一版不做技能成长

当前角色已有基础技能，但策划稿中的：

```text
每个职业 3 个技能分支
每代只选 1 个主动技能
后续两次强化该技能
```

第一版暂时不做。

原因：

- 当前角色和 Boss 已经有不少待打磨内容。
- 加点、武器升级、服务节点、多结局、存档已经足够构成第一版主循环。
- 技能成长需要大量 UI、数值和角色脚本配合，容易拖慢整合。

第一版技能目标改为：

```text
保留现有角色技能
保证技能能正常释放、伤害正确、UI 冷却正确
不做技能分支选择
不做技能强化树
```

后续有余力再作为第二版扩展。

## 2. 消耗品槽

### 2.1 系统目标

玩家应拥有 1 个主动消耗品槽。

规则：

```text
同一时间只能装备 1 种消耗品
同类消耗品可堆叠数量
异类消耗品需要弹出替换/放弃面板
死亡或进入下一代后不继承
```

### 2.2 消耗品来源

消耗品可以来自：

```text
军需库免费装备/补给
商店购买
地图小物件/宝箱掉落
教堂或特殊节点奖励
Boss 后补给
```

注意：消耗品不应该每关强制给，应该作为有限资源。

### 2.3 第一版消耗品类型

建议第一版只做 4-6 个。

| 消耗品 | 效果 | 类型 |
|---|---|---|
| 医疗瓶 | 恢复 25% 最大生命 | 瞬发 |
| 护甲包 | 恢复护甲，或获得少量护盾 | 瞬发 |
| 磨锋油 | 8 秒内攻击伤害 +15% | 短时状态 |
| 疾步药剂 | 8 秒内移动速度 +18% | 短时状态 |
| 聚灵药剂 | 立即恢复灵感，短时间技能冷却略降 | 瞬发/短时状态 |
| 烟幕瓶 | 短时间降低敌人追踪或创造逃生窗口 | 功能型，可选 |

### 2.4 使用规则

建议：

```text
瞬发型：按键后立即生效并扣数量
短施放型：生效瞬间扣数量，被打断则不扣
短时状态型：生效后显示持续时间
```

第一版可以先全部做成瞬发或短时状态，不做复杂打断。

### 2.5 UI 需求

HUD 需要显示：

```text
当前消耗品图标
数量
是否可用
冷却/持续状态
空槽提示
```

拾取异类时需要显示替换面板：

```text
保留当前
替换为新消耗品
放弃新消耗品
```

### 2.6 建议代码结构

建议新增：

```text
systems/consumables/consumable_manager.gd
systems/consumables/consumables.json
ui/consumable_slot.gd
ui/consumable_replace_panel.gd
```

核心 API：

```gdscript
func reset_generation() -> void
func get_state() -> Dictionary
func can_pickup(consumable_id: String) -> bool
func pickup(consumable_id: String, amount: int = 1) -> Dictionary
func replace_with(consumable_id: String, amount: int = 1) -> void
func use_current(actor: Node) -> bool
func apply_effect(consumable_id: String, actor: Node) -> void
```

### 2.7 验收标准

- HUD 能显示当前消耗品槽。
- 玩家能拾取消耗品。
- 同类会堆叠。
- 异类会弹替换面板。
- 使用后数量减少。
- 数量为 0 后槽位清空。
- 死亡进入下一代后消耗品清空。

## 3. 存档复杂度扩展

### 3.1 为什么需要扩展

后续加入王冠循环、多结局、皇帝轮替、血脉火种、资质、武器升级、服务节点后，存档不能只保存一个简单 run 状态。

存档需要能恢复：

```text
当前大周目/王冠循环
当前第几代
火种数量
当前皇帝是谁
哪些家族已经当过皇帝
哪些家族还可选
当前皇帝血量
当前章节/房间进度
当前资质
当前金币和经济状态
当前饰品
当前武器升级
当前消耗品
服务节点使用状态
已触发结局条件
```

### 3.2 建议存档分层

建议把存档拆成几块，而不是所有字段平铺。

```gdscript
{
	"version": 2,
	"slot_meta": {},
	"lineage": {},
	"throne_cycle": {},
	"run": {},
	"build": {},
	"map": {},
	"ending_flags": {}
}
```

### 3.3 `lineage`

保存血脉和代际：

```gdscript
{
	"family_id": "knight",
	"generation_index": 2,
	"seeds_left": 4,
	"max_seeds": 5,
	"aptitude_total": 12,
	"aptitude": {
		"strength": 5,
		"agility": 4,
		"focus": 3
	},
	"last_score": {
		"grade": "A",
		"score": 78
	}
}
```

### 3.4 `throne_cycle`

保存王冠循环：

```gdscript
{
	"cycle_index": 2,
	"current_emperor_family": "mage",
	"crowned_families": ["knight"],
	"available_families": ["ranger", "mage"],
	"prince_defeated": true
}
```

### 3.5 `run`

保存当前局内状态：

```gdscript
{
	"current_encounter_index": 5,
	"cleared_encounters": 5,
	"gold": 72,
	"hero_level": 3,
	"hero_xp": 18,
	"total_kills": 24,
	"event_history": []
}
```

### 3.6 `build`

保存构筑：

```gdscript
{
	"accessory_id": "wolf_pendant",
	"weapon_upgrades": ["weapon_damage_15", "weapon_speed_10"],
	"consumable": {
		"id": "healing_potion",
		"count": 2
	},
	"run_modifiers": {}
}
```

注意：

- 武器升级和消耗品是本代内容。
- 死亡进入下一代后应清空。
- 如果设计为断线恢复当前代，则存档内仍要保存当前代状态。

### 3.7 `map`

保存地图和服务节点：

```gdscript
{
	"chapter_id": "town",
	"room_index": 5,
	"used_service_nodes": {
		"church_01": true,
		"shop_01": false,
		"armory_01": true
	}
}
```

### 3.8 `ending_flags`

保存结局条件：

```gdscript
{
	"can_break_crown": false,
	"can_escape": true,
	"ember_extinguished": false,
	"wear_crown_bad_ready": false
}
```

### 3.9 版本迁移

存档需要 `version` 字段。

```text
version = 1：当前旧存档
version = 2：加入王冠循环、消耗品、武器升级后的新存档
```

读取旧存档时要补默认值，不能直接报错。

### 3.10 验收标准

- 退出重进后，火种、代数、资质能恢复。
- 退出重进后，当前皇帝家族、已登基家族、可选家族能恢复。
- 退出重进后，皇帝剩余血量能恢复。
- 退出重进后，武器升级、饰品、消耗品能恢复当前代状态。
- 死亡进入下一代后，武器升级和消耗品清零。
- 服务节点使用状态能保存。
- 旧存档不会导致游戏崩溃。
