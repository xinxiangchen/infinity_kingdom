# 角色组武器升级接口约定

更新日期：2026-06-12

本文档给角色组使用。武器升级的具体表现、动画、判定、数值落点由角色组负责；整合组只负责在 Boss 奖励、商店、军需库等系统中触发升级，并通过统一 API 通知当前角色。

目标是让角色模块保持封装：外部系统不直接修改角色的攻击力、攻速、武器节点、动画播放器或碰撞箱。

## 1. 分工边界

### 1.1 角色组负责

角色组需要负责：

- 角色武器升级状态的保存与清空。
- 武器升级对普攻伤害、攻击间隔、挥砍判定、投射物、武器动画的实际影响。
- 不同角色或 Boss 的武器升级表现差异。
- 武器升级后 UI 需要读取的展示状态。
- 轮回清空时，角色能恢复到未升级状态。

角色组可以自行决定内部实现方式，例如：

- 直接在角色基类里维护升级状态。
- 做一个 `WeaponUpgradeState` 子组件。
- 每个角色独立实现，但对外 API 保持一致。

### 1.2 整合组负责

整合组只负责：

- 在打过收官 Boss 后，弹出或触发武器升级选择。
- 在商店中提供花钱升级武器的入口。
- 在军需库中提供升级武器或领取装备的入口。
- 调用角色暴露的武器升级 API。
- 在角色死亡、轮回、重新开局时调用清空 API。
- 存档时读取角色暴露的升级状态，读档时把状态交还给角色恢复。

整合组不应该：

- 直接改 `player.damage`。
- 直接改 `player.attack_interval`。
- 直接改角色内部武器节点的 scale、rotation、collision shape。
- 直接操作角色内部动画树或动画播放器。
- 根据角色类型写大量 `if knight / ranger / mage` 的武器逻辑。

## 2. 必须暴露的角色 API

每个可操作角色至少需要提供以下方法。

```gdscript
func can_accept_weapon_upgrade(upgrade_id: String, upgrade_data: Dictionary) -> bool
```

用途：整合组在展示升级选项前，询问当前角色是否能接受该升级。

返回：

- `true`：可以选择。
- `false`：不应该展示，或展示为不可选。

```gdscript
func apply_weapon_upgrade(upgrade_id: String, upgrade_data: Dictionary) -> void
```

用途：玩家确认选择后，整合组调用这个方法，把升级交给角色内部处理。

要求：

- 角色内部记录该升级已经获得。
- 角色内部更新伤害、攻速、动画或判定需要用到的倍率。
- 重复升级是否允许，由角色内部决定，但需要和策划规则一致。

```gdscript
func clear_weapon_upgrades() -> void
```

用途：角色死亡进入下一代、重新开始大周目、或调试模式重置角色时调用。

要求：

- 清空本代获得的所有武器升级。
- 恢复基础伤害、基础攻击间隔、基础武器表现。
- 不影响角色的资质、等级、饰品、消耗品等其他系统。

```gdscript
func get_weapon_upgrade_state() -> Dictionary
```

用途：UI、存档、调试面板读取当前武器升级状态。

建议返回格式：

```gdscript
{
	"applied_upgrades": ["weapon_damage_15", "weapon_speed_10"],
	"damage_multiplier": 1.15,
	"attack_interval_multiplier": 0.90
}
```

```gdscript
func restore_weapon_upgrade_state(state: Dictionary) -> void
```

用途：读档或调试恢复状态时调用。

要求：

- 不要只恢复最终倍率。
- 推荐根据 `applied_upgrades` 重新应用升级数据。
- 这样后续数值表改动时，不容易出现旧倍率残留。

## 3. 武器升级数据约定

整合组传给角色的 `upgrade_data` 建议保持以下结构。

```gdscript
{
	"id": "weapon_damage_15",
	"name": "锋刃强化 I",
	"summary": "武器伤害 +15%",
	"type": "weapon_upgrade",
	"effects": {
		"attack_damage_multiplier": 1.15
	}
}
```

```gdscript
{
	"id": "weapon_speed_10",
	"name": "轻巧磨锋",
	"summary": "攻击速度 +10%",
	"type": "weapon_upgrade",
	"effects": {
		"attack_interval_multiplier": 0.90
	}
}
```

第一版只需要支持四种升级：

| id | 效果 | 角色内部建议解释 |
|---|---|---|
| `weapon_speed_10` | 攻速 +10% | 普攻间隔乘以 `0.90` |
| `weapon_speed_15` | 攻速 +15% | 普攻间隔乘以 `0.85` |
| `weapon_damage_15` | 伤害 +15% | 武器/普攻伤害乘以 `1.15` |
| `weapon_damage_30` | 伤害 +30% | 武器/普攻伤害乘以 `1.30` |

注意：攻速提升不要写成冷却时间加法，建议统一使用攻击间隔倍率。

## 4. 角色内部实现要求

### 4.1 不要只改显示数值

武器升级必须真的影响战斗逻辑，而不只是 HUD 文本。

至少需要作用到：

- 普攻实际伤害。
- 普攻实际冷却或攻击间隔。
- 近战武器的判定生成时机。
- 远程武器的投射物发射频率或投射物伤害。

### 4.2 挥砍判定和动画要一起调整

如果角色是持剑、长枪、斧头等近战角色，武器升级后不能只改数值。

角色组需要检查：

- 武器挥砍动画轨迹是否和伤害判定一致。
- 武器旋转中心是否稳定。
- 武器是否出现异常自转。
- 左右移动时，角色贴图和武器贴图是否一起转向。
- 攻速提高后，判定帧是否仍然对齐动画。

### 4.3 不同角色可以有不同表现

同一个升级在不同角色上可以表现不同，但最终数值语义要一致。

例如：

- 骑士：武器变重，挥砍范围略明显，剑的视觉尺寸可以更突出。
- 猎鹰者：攻击频率提升可以体现在发射节奏或武器前摇缩短。
- 法师：如果普攻不是剑，可以作用到法器/弹体伤害或发射间隔。

但整合组不需要知道这些差异，只调用统一 API。

## 5. 和其他系统的关系

### 5.1 和饰品不同

武器升级不是饰品。

它不应该放进饰品管理器，也不应该占用饰品栏。

### 5.2 和状态奖励不同

武器升级不是普通状态奖励。

它不应该混进每关随机三选一的状态池，例如 `forge_edge`、`forge_focus`、`forge_guard` 这类旧奖励后续需要区分处理。

### 5.3 和轮回的关系

武器升级只属于当前这一代。

角色死亡进入下一代时：

```gdscript
player.clear_weapon_upgrades()
```

必须清空。

资质、火种、皇帝血量推进可以跨代；武器升级不跨代。

### 5.4 和存档的关系

存档系统只保存角色暴露的状态：

```gdscript
var state := player.get_weapon_upgrade_state()
```

读档时：

```gdscript
player.restore_weapon_upgrade_state(state)
```

存档系统不直接计算倍率。

## 6. 整合组调用流程

### 6.1 Boss 奖励

```text
打过收官 Boss
-> 生成 3 个武器升级选项
-> 根据 player.can_accept_weapon_upgrade 过滤
-> 玩家选择 1 个
-> player.apply_weapon_upgrade(upgrade_id, upgrade_data)
```

### 6.2 商店

```text
进入商店
-> 展示 1 到 2 个武器升级商品
-> 玩家支付金币
-> player.apply_weapon_upgrade(upgrade_id, upgrade_data)
```

### 6.3 军需库

```text
进入军需库
-> 玩家在免费装备、补给、武器升级之间选择
-> 如果选择武器升级
-> player.apply_weapon_upgrade(upgrade_id, upgrade_data)
```

### 6.4 轮回

```text
玩家死亡
-> 结算评分
-> 进入下一代
-> player.clear_weapon_upgrades()
-> 生成下一代资质
```

## 7. 验收标准

角色组提交时至少需要满足：

- 当前三个玩家角色都实现统一武器升级 API。
- 四种第一版升级都能被角色接受并产生真实战斗效果。
- 攻速升级会改变实际攻击间隔，不只是改变动画速度。
- 伤害升级会改变实际命中的伤害。
- 死亡轮回后升级清空。
- 左右移动时角色和武器贴图能正确翻转。
- 近战挥砍动画和碰撞判定轨迹基本一致。
- 调试入口能直接给角色施加武器升级，并看见效果。

## 8. 给角色组的实现建议

推荐角色组内部维护以下状态：

```gdscript
var weapon_upgrade_ids: Array[String] = []
var weapon_damage_multiplier: float = 1.0
var weapon_attack_interval_multiplier: float = 1.0
```

应用升级时：

```gdscript
func apply_weapon_upgrade(upgrade_id: String, upgrade_data: Dictionary) -> void:
	if not can_accept_weapon_upgrade(upgrade_id, upgrade_data):
		return
	weapon_upgrade_ids.append(upgrade_id)
	var effects := upgrade_data.get("effects", {})
	weapon_damage_multiplier *= float(effects.get("attack_damage_multiplier", 1.0))
	weapon_attack_interval_multiplier *= float(effects.get("attack_interval_multiplier", 1.0))
	_refresh_weapon_stats()
```

攻击计算时：

```gdscript
var final_damage := base_weapon_damage * weapon_damage_multiplier
var final_interval := base_attack_interval * weapon_attack_interval_multiplier
```

这只是建议写法，不强制。强制要求是：对外 API 一致，外部系统不需要知道角色内部怎么实现。
