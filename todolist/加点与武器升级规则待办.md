# 加点与武器升级规则待办

更新日期：2026-06-11

本文档用于固定后续要改的两个核心系统：代际加点规则、武器升级规则。当前代码中这两块都只是占位或半成品，后续实现时以本文档为准。

## 1. 当前问题

### 1.1 加点系统问题

当前代码里第一代开局没有加点，第一代固定为：

```text
强壮 3 / 迅捷 3 / 专注 3
```

死亡后，系统根据上一代评分生成下一代资质总点数，然后玩家再额外选择一项 `+1`。这会导致几个问题：

- 玩家开局没有构筑参与感。
- 加点只出现在死亡后，像死亡惩罚结算，不像角色构筑。
- 当前手动加点是额外白送 `+1`，可能突破总点数上限。
- 评分、随机分配、手动加点之间的关系不清楚。

### 1.2 武器升级系统问题

当前代码里有 `forge_edge`、`forge_focus`、`forge_guard` 等锻造奖励，但它们本质上是状态奖励或本局 modifier，不是真正的武器升级系统。

后续需要单独实现“武器升级系统”，不要再用普通关间事件替代。

## 2. 加点系统新规则

### 2.1 基础规则

每一代都有三项资质：

```text
强壮 strength
迅捷 agility
专注 focus
```

第一代开局应有加点/微调入口，而不是直接 3/3/3 进游戏。

建议第一代基础总点数：

```text
第一代：9 点
```

第一代默认分配可以按职业偏向：

```text
骑士：强壮 4 / 迅捷 3 / 专注 2
游侠：强壮 2 / 迅捷 4 / 专注 3
法师：强壮 2 / 迅捷 3 / 专注 4
```

玩家开局可以微调 1 点：

```text
从一个属性 -1，给另一个属性 +1
```

注意：微调不是额外加点，不改变总点数。

### 2.2 死亡后的评分修正规则

每次死亡后，根据上一代评分修正下一代总点数。

评分修正：

```text
S：下一代总点数 +3
A：下一代总点数 +2
B：下一代总点数 +1
C：下一代总点数 +0
D：下一代总点数 -1
```

总点数上下限：

```text
最低：7 点
最高：15 点
```

示例：

```text
第一代 9 点，S 评分死亡 -> 第二代 12 点
第二代 12 点，S 评分死亡 -> 第三代 15 点
第三代 15 点，S 评分死亡 -> 第四代仍然 15 点
```

```text
第一代 9 点，A 评分死亡 -> 第二代 11 点
第二代 11 点，B 评分死亡 -> 第三代 12 点
第三代 12 点，C 评分死亡 -> 第四代 12 点
第四代 12 点，D 评分死亡 -> 第五代 11 点
```

### 2.3 四次轮回与火种

如果初始火种为 5，则玩家最多经历：

```text
第一代 + 4 次死亡后的继承者
```

也就是最多能通过四次死亡继承来调整资质。

如果每次死亡评分都是 S：

```text
第一代：9
第二代：12
第三代：15
第四代：15
第五代：15
```

不会突破 15 点。

### 2.4 下一代点数分配方式

下一代总点数确定后，可以按职业权重自动分配初稿：

```text
骑士偏强壮
游侠偏迅捷
法师偏专注
```

然后玩家进行一次微调：

```text
从一个属性挪 1 点到另一个属性
```

不再使用“直接 +1”的方式。

### 2.5 评分和加点的耦合关系

新的耦合关系应改成：

```text
上一代死亡表现 -> 评分等级 -> 下一代总点数变化 -> 自动分配 -> 玩家微调
```

不要再让评分直接随机生成一个完全独立的总点数。

### 2.6 代码改动目标

需要改的核心位置：

```text
systems/run/lineage_director.gd
ui/heir_select_panel.gd
world.gd
ui/lineage_hud.gd
```

建议新增或调整字段：

```gdscript
var aptitude_total: int = 9
var current_aptitude := {
	"strength": 4,
	"agility": 3,
	"focus": 2
}
```

建议新增方法：

```gdscript
func grade_to_point_delta(grade: String) -> int
func generate_aptitude_from_total(total: int, family_id: String) -> Dictionary
func apply_manual_aptitude_shift(from_attribute: String, to_attribute: String) -> Dictionary
func get_aptitude_total() -> int
```

需要替换当前逻辑：

```gdscript
apply_manual_aptitude_bonus(attribute_id)
```

这个方法现在是直接 `+1`，后续应替换成“从一项挪到另一项”的微调逻辑。

## 3. 武器升级系统新规则

### 3.1 触发时机

武器升级不作为每关奖励随机出现，而是固定关键节点奖励。

当前确定两个固定来源：

```text
1. 打过收官 Boss 后固定获得一次武器升级
2. 商店中可花钱购买一次武器升级机会
3. 军需库中可选择武器升级或免费装备
```

说明：

- “收官 Boss”指阶段性关键 Boss，例如城镇守关 Boss 或当前章节的守关 Boss。
- 商店中的武器升级应该是明确商品，不是随机普通状态奖励。
- 军需库是空间交互节点，提供免费装备或武器升级类军备选项。
- 武器升级不应混在 `bounty / pact / attunement / scout` 这种普通关间事件池里。

### 3.2 轮回后清零

武器升级只属于当前这一代。

规则：

```text
角色死亡并进入下一代后，武器升级全部清零。
```

原因：

- 轮回继承的核心应该是资质和皇帝血量推进，不是武器词条永久继承。
- 防止多代叠加后数值爆炸。
- 让每一代都有重新构筑武器路线的空间。

### 3.3 升级类型

先做最小可实现版本，只包含攻速和伤害两类数值升级。

武器升级选项：

```text
攻速 +10%
攻速 +15%
伤害 +15%
伤害 +30%
```

建议命名：

```text
轻巧磨锋：攻速 +10%
疾速铭刻：攻速 +15%
锋刃强化：伤害 +15%
王印重铸：伤害 +30%
```

### 3.4 数值含义

攻速提升：

```text
攻速 +10% = 普攻间隔 * 0.90
攻速 +15% = 普攻间隔 * 0.85
```

伤害提升：

```text
伤害 +15% = 武器/普攻伤害 * 1.15
伤害 +30% = 武器/普攻伤害 * 1.30
```

如果当前角色有技能也使用武器伤害，可以先只作用于普攻，后续再决定是否影响技能。

### 3.5 升级选择方式

最小实现建议：

```text
每次触发武器升级时，从 4 个升级中展示 3 个，玩家选择 1 个。
```

也可以商店固定展示 2 个：

```text
商店：展示 2 个武器升级，需要花金币购买
收官 Boss：展示 3 个武器升级，免费选择 1 个
```

具体 UI 可以复用现有 `run_event_panel` 的三选一形式，但需要在代码层面单独区分“武器升级”与“普通状态奖励”。

### 3.6 叠加规则

当前最小规则：

```text
同一代最多获得 2 次武器升级
升级可以叠加
死亡后清零
```

叠加示例：

```text
选择伤害 +15%，再选择伤害 +30%
最终普攻伤害倍率 = 1.15 * 1.30 = 1.495
```

```text
选择攻速 +10%，再选择攻速 +15%
最终普攻间隔倍率 = 0.90 * 0.85 = 0.765
```

如后续觉得数值过强，可以改成同类取最高或做递减收益，但第一版先保持简单。

### 3.8 和现有系统的关系

需要注意：

- 武器升级不是饰品。
- 武器升级不是 `RunEffects` 中的普通事件奖励。
- 武器升级不应该被 `AccessoryManager.reset_run()` 管理。
- 轮回后武器升级清零，应在下一代重生时调用 `WeaponUpgradeManager.reset_generation()`。
- 如果普通重新开始整个大周目，也要清零。

可以临时复用 UI，但系统层要独立。

## 4. 关间奖励节奏调整建议

当前每过一关都给饰品或状态奖励过于频繁。后续建议改成：

```text
普通小怪关：金币/经验/少量掉落
商店：固定可买武器升级，另有少量补给或饰品
收官 Boss：固定武器升级
关键剧情/皇帝门前：可给饰品、恢复、最终准备
死亡轮回：资质变化，武器升级清零
```

这样能让奖励职责更清楚：

```text
资质：跨代成长
武器：本代战斗成长
饰品：本代构筑偏向
消耗品：短期战术资源
金币：商店购买资源
```

## 5. 验收标准

### 5.1 加点系统验收

- 第一代开局能看到资质分配或微调界面。
- 每代资质总点数始终在 7-15。
- S/A/B/C/D 评分分别对应 +3/+2/+1/+0/-1。
- 玩家微调不会改变总点数。
- HUD 能显示当前代数、火种、三项资质和总点数。

### 5.2 武器升级系统验收

- 打过收官 Boss 后固定出现武器升级选择。
- 商店固定提供武器升级购买入口。
- 攻速 +10%、攻速 +15%、伤害 +15%、伤害 +30% 能正确作用于角色。
- 死亡进入下一代后武器升级清零。
- 武器升级不再混入普通每关事件奖励。
