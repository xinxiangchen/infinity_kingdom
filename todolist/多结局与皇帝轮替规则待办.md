# 多结局与皇帝轮替规则待办

更新日期：2026-06-11

本文档用于固定后续主线循环、结局和皇帝轮替规则。当前代码中，最终遭遇打完会直接进入胜利/结局结果页；后续设计应改成：打败王子后不立刻结束游戏，而是进入下一轮皇帝循环。

## 1. 当前代码状态

当前 `world.gd` 中，最终遭遇胜利后会调用：

```gdscript
_complete_run_victory()
```

这个方法会：

```text
记录最终 Boss 被击败
判断结局类型
显示胜利/结局结果页
返回选择角色或结束当前流程
```

当前已有的结局判断雏形：

```text
普通胜利 victory
打碎王冠 true_ending
开发者房间 developer_room
火种耗尽 defeat/archive burned out
```

当前已有的最终 Boss 选择雏形：

```text
第一轮 reincarnation <= 1：双子/王子 Boss
后续轮回：根据当前 family_id 选择骑士/游侠/法师对应最终 Boss
```

但这还不是完整设计。当前问题是：

- 打过王子后流程会进入胜利页，像是游戏结束。
- 没有把王子作为“下一轮皇帝/下一轮 Boss 循环”的起点。
- 没有记录三个家族是否都当过皇帝。
- 没有根据已登基家族限制玩家后续可选角色。
- 没有根据当前皇帝家族改变小兵刷新比例。

## 2. 新主线循环目标

后续主线应改成：

```text
第一轮：三家族可选 -> 打王子/双子
打败王子后：不结束游戏，进入下一轮
下一轮：上一轮选择/结局导致某个家族登基成为皇帝
玩家只能从剩余家族中选角色
继续挑战新皇帝
再打败后继续轮替
直到触发某个结局
```

也就是说，王子不是最终结局，而是第一轮的关键 Boss。

## 3. 多结局设计

### 3.1 打碎王冠

结局定位：

```text
真结局 / 终止轮回
```

触发方向：

- 玩家满足特定条件后，最终选择打碎王冠。
- 打碎王冠后，王位循环结束。
- 这是最理想结局。

当前代码已有 `true_ending / Crown Breaker` 的雏形，后续应保留并重写触发条件。

### 3.2 逃向远方

结局定位：

```text
中性结局 / 放弃王冠
```

触发方向：

- 玩家在某次击败皇帝后选择不继承、不打碎王冠，而是逃离王国。
- 可以作为一种提前结束路线。
- 不一定是失败，但也没有解决王冠循环。

需要新增：

```text
ending_kind = "escape"
```

### 3.3 火种熄灭

结局定位：

```text
失败结局
```

触发方向：

- 血脉火种耗尽。
- 皇帝仍未被彻底解决。
- 当前档案结束。

当前代码已有火种耗尽失败结果页雏形，后续应把文案和结局类型统一到结局系统中。

建议：

```text
ending_kind = "ember_extinguished"
```

### 3.4 戴上王冠

结局定位：

```text
坏结局
```

触发方向：

- 经历四次轮回后，三个家族都曾经当过皇帝。
- 玩家仍然选择戴上王冠。
- 王冠循环没有被打破，只是换了新的统治者。

这是 bad ending。

建议：

```text
ending_kind = "wear_crown_bad"
```

文案方向：

```text
三个家族都证明过自己无法摆脱王冠。
继承者终于坐上王座，却只是成为下一任囚徒。
```

## 4. 皇帝轮替规则

### 4.1 家族登基记录

需要记录哪些家族已经当过皇帝：

```gdscript
var crowned_families: Array[String] = []
```

可能值：

```text
knight
ranger
mage
```

每打完一轮并进入下一轮时，把对应家族加入 `crowned_families`。

### 4.2 可选角色限制

第一轮：

```text
骑士 / 游侠 / 法师 都可选
```

打过王子后进入下一轮：

```text
只能从剩余两个家族中选择
```

再打过下一任皇帝后：

```text
只能从最后一个未登基家族中选择
```

如果三个家族都当过皇帝：

```text
进入最终结局分歧
```

### 4.3 当前皇帝家族

需要维护：

```gdscript
var current_emperor_family: String = ""
```

它决定：

- 当前最终 Boss 使用哪个家族形态。
- 普通小兵刷新权重。
- 关卡文案和音乐倾向。
- 后续结局条件。

## 5. 小兵刷新权重

当前想法：

```text
当前皇帝对应家族的小兵，占比 50%
其余两大家族的小兵，共占 50%
```

建议第一版：

```text
皇帝家族小兵：50%
其他家族 A：25%
其他家族 B：25%
```

示例：

如果当前皇帝是法师：

```text
法师系小兵：50%
骑士系小兵：25%
游侠系小兵：25%
```

如果当前皇帝是骑士：

```text
骑士系小兵：50%
游侠系小兵：25%
法师系小兵：25%
```

如果当前皇帝是游侠：

```text
游侠系小兵：50%
骑士系小兵：25%
法师系小兵：25%
```

## 6. Boss 轮替建议

第一轮：

```text
王子/双子 Boss
```

后续轮次：

```text
骑士家族登基 -> 骑士皇帝 Boss
游侠家族登基 -> 游侠皇帝 Boss
法师家族登基 -> 法师皇帝 Boss
```

注意：打败王子之后不应该显示最终结局，而应进入下一轮选择。

## 7. 需要新增或改造的系统

建议新增：

```text
systems/run/throne_cycle_director.gd
```

职责：

- 记录当前王冠循环进度。
- 记录已经登基的家族。
- 记录当前皇帝家族。
- 给角色选择 UI 提供可选家族列表。
- 给敌人生成系统提供小兵权重。
- 给最终 Boss 选择逻辑提供 Boss 类型。
- 判断是否进入结局分歧。

建议 API：

```gdscript
func start_cycle() -> void
func get_available_families() -> Array[String]
func choose_challenger_family(family_id: String) -> void
func record_prince_defeated(winning_family: String) -> void
func record_emperor_defeated(winning_family: String) -> void
func get_current_emperor_family() -> String
func get_enemy_family_weights() -> Dictionary
func should_enter_final_choice() -> bool
func choose_ending(choice_id: String) -> Dictionary
func get_state() -> Dictionary
```

## 8. 需要改的现有位置

### 8.1 `world.gd`

需要调整：

- `_complete_run_victory()` 不应总是进入结果页。
- 打败王子后应调用王冠循环推进，而不是结束游戏。
- 最终 Boss 选择应由 `ThroneCycleDirector` 决定。

### 8.2 `ui/character_select.gd`

需要调整：

- 根据 `get_available_families()` 禁用已登基家族。
- UI 明确说明为什么当前只能选两个或一个角色。

### 8.3 敌人生成系统

需要调整：

- 普通敌人按当前皇帝家族权重生成。
- 当前皇帝家族对应小兵占 50%。

### 8.4 `EndingDirector`

需要调整：

- 结局不只由无伤/教堂洗礼决定。
- 新增打碎王冠、逃向远方、火种熄灭、戴上王冠 bad ending。
- 将结局条件和王冠循环状态关联。

## 9. 结局选择点

当满足条件时，应该出现结局选择 UI，而不是自动结算。

可能选项：

```text
打碎王冠
逃向远方
戴上王冠
```

如果火种耗尽，则不选择，直接进入火种熄灭结局。

## 10. 验收标准

- 打败王子后不结束游戏，而是进入下一轮。
- 下一轮只能选择剩余两个家族。
- 再打败一任皇帝后，只能选择最后一个未登基家族。
- 三个家族都当过皇帝后，进入最终结局分歧。
- 当前皇帝家族对应小兵刷新比例为 50%。
- 火种耗尽触发火种熄灭结局。
- 可以触发打碎王冠结局。
- 可以触发逃向远方结局。
- 可以触发戴上王冠 bad ending。
